### 2.1 기능 완성도 — A
- 수업일정표/강의안: 15회차 일정·강의 노트(강사 PT 제작 가능 수준).
- 개인별 PBL 활동: 6단계 작성 + 자동 채점(0~100) + 강사 평가 + 명단 대조 + 회원정보 반영.
- 자가평가: 사전·진단·사후(20문항×3), 번호 네비·즉시 채점·응시 3회/자습.
- 프로젝트: 아이디어 예시·구현 가이드·자체 팀구성·게시판.
- LMS/관리자: 공지·자료·과제·Q&A·명단·평가 화면.

### 2.2 콘텐츠 — A
- **강의 노트**: 학술/기술 용어 기반 전문 서술(EDA·RAG·인과추론 등), 핵심 포인트·예시 포함.
- **프로젝트 14주제**: 지역·난이도·트랙·추천 데이터·산출물·확장까지 구조화.
- **평가 60문항**: ESG·데이터·생성형 AI·PBL 전 범위 객관식 + 해설.

### 2.3 반응형 — A (실측)
- 390px(모바일) 기준 홈·강의안·프로젝트·평가 페이지 **가로 스크롤 0** 확인.
- 사이드바 페이지 모바일 세로 스택, 넓은 표(루브릭) 가로 스크롤 래퍼, PBL 점수 행 반응형 처리.

### 2.4 데이터/보안 — A / B+
- Supabase **snu_ 접두사**로 공용 프로젝트 내 격리, 전 테이블 RLS, AuthGuard/AdminGuard.
- 권고: 자가평가/PBL 점수의 학생 자가수정 가능성(트러스트 모델) — 엄정 평가 시 점수 컬럼 분리·서버검증 검토.

## 3. 강점
1. **자동 평가 + 콘텐츠 영구 저장** — koreatech 대비 학생 작성 내용 보존(개선점 해결).
2. **명단 대조** — 소속(대학·학과·전공)·트랙으로 수강 명단 자동 확인.
3. **강사 PT용 강의 노트** — 11회차 전문 강의안 즉시 활용.
4. **운영 자동화** — 푸시 한 번으로 빌드·배포, 멱등 DB 스키마.
